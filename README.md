# ConexyTask

**ConexyTask** — это минималистичное, но функциональное веб-приложение для управления задачами (To-Do лист) со встроенным таймером. Проект разработан как монолит, состоящий из бэкенда на ASP.NET Core Web API и одностраничного фронтенда (SPA) на чистом JavaScript. Особенность проекта — гибридное хранение данных: основная информация о задачах сохраняется в PostgreSQL, а расширенные параметры (приоритеты, таймеры) — в `sessionStorage` браузера, что позволяет приложению работать даже при недоступности сервера, а при закрытии вкладки данные автоматически очищаются.

> 🔗 **Попробовать**: https://conexytask.onrender.com

---

## 🚀 Ключевые возможности

### Управление задачами
- Создание, редактирование и удаление задач.
- Автоматический расчет приоритета (Красный 🔴, Желтый 🟡, Зеленый 🟢).
- Визуальная подсветка всей карточки задачи в соответствии с приоритетом.

### Гибкие настройки
- Установка сложности (1–3) и срочности (1–3).
- Выбор дедлайна через встроенный календарь.
- Тип повторения: ежедневно, еженедельно, ежемесячно.

### Умные повторения
- При выполнении повторяющейся задачи автоматически создаётся её копия с новой датой.

### Встроенный таймер
- Для каждой задачи можно запустить таймер обратного отсчета.
- Возможность перехода в полноэкранный режим для максимальной концентрации.
- Таймер продолжает работать даже при переключении между задачами.

### Автономный режим (Fallback)
- Если сервер недоступен, все операции выполняются в `sessionStorage` браузера.
- При перезагрузке страницы данные восстанавливаются, при закрытии вкладки — полностью очищаются.

### Адаптивный интерфейс
- Стильный дизайн в стиле "стекло" (glassmorphism) корректно отображается на всех устройствах.
- Полная поддержка мобильных телефонов и планшетов.

### Чистая архитектура
- Бэкенд разделен на четкие слои (Controller, Service, Repository), что облегчает поддержку и масштабирование.

## 📁 Архитектура проекта

```
ConexyTask/
├── Controllers/               # API эндпоинты (ConexyController)
├── DbContext/                  # Контекст базы данных (DbConexy)
├── Entity/                      # Сущности БД (ConexyEntity)
├── Model/                        # Бизнес-модели (Conexy)
├── Repository/                   # Репозитории (ConexyRepository, IConexyRepository)
├── Service/                      # Сервисы (ConexyService, IConexyService)
├── Configuration/                # Конфигурации Entity (Conexy_config)
├── Contract/                      # DTO для запросов/ответов
├── Properties/                    # Настройки запуска (launchSettings.json)
├── wwwroot/                        # Статические файлы (фронтенд)
│   └── index.html                   # Главная SPA-страница
├── Program.cs                       # Точка входа, DI, middleware
├── appsettings.json                 # Основная конфигурация (БД, логи)
├── appsettings.Development.json     # Конфигурация для разработки
└── ConexyTask.csproj                # Файл проекта
```

### Схема взаимодействия
1. Фронтенд (`index.html`) общается с бэкендом через REST API.
2. Бэкенд обрабатывает запросы, используя сервисы и репозитории.
3. Основные данные (название, описание) хранятся в PostgreSQL.
4. Дополнительные поля (сложность, срочность, дедлайн, таймер) хранятся в `sessionStorage` браузера.
5. При недоступности сервера все операции переключаются на локальное хранилище.

## 🔌 API Endpoints

| Метод | URL | Описание | Тело запроса |
|-------|-----|----------|--------------|
| GET | `/api/tasks` | Получить все задачи | – |
| POST | `/api/tasks` | Создать новую задачу | `{ "name": "string", "description": "string" }` |
| PUT | `/api/tasks/{id}` | Обновить задачу | `{ "name": "string", "description": "string" }` |
| DELETE | `/api/tasks/{id}` | Удалить задачу | – |

> ⚠️ Примечание: поля `complexity`, `urgency`, `deadline` и `timer` не отправляются на сервер, а хранятся локально. Это позволяет расширять функционал без изменения схемы БД.

## 🌍 Переменные окружения

Для деплоя на Render используются следующие переменные:

| Переменная | Описание | Пример |
|------------|----------|--------|
| `ASPNETCORE_ENVIRONMENT` | Среда выполнения | `Production` |
| `PORT` | Порт, на котором слушает приложение | `8080` (задаётся платформой) |
| `ConnectionStrings__DefaultConnection` | Строка подключения к PostgreSQL | `Host=...;Database=...;Username=...;Password=...` |

В `Program.cs` реализована автоматическая конвертация `postgres://` URL в формат Npgsql, что упрощает деплой на Render.

## 🚀 Локальный запуск

### Требования
- [.NET 9.0 SDK](https://dotnet.microsoft.com/download)
- [PostgreSQL](https://www.postgresql.org/download/) (или Docker)
- Git

### Шаги

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/rem1x_zov/conexytask.git
   cd conexytask
   ```

2. **Настройте базу данных**
   - Создайте базу данных (например, `conexy_db`).
   - В файле `appsettings.json` укажите строку подключения:
     ```json
     "ConnectionStrings": {
       "DefaultConnection": "Host=localhost;Port=5432;Database=To_Do_Flip;Username=postgres;Password=123456"
     }
     ```

3. **Примените миграции**
   ```bash
   dotnet ef database update
   ```
   *Если `dotnet ef` не установлен: `dotnet tool install --global dotnet-ef`*

4. **Запустите приложение**
   ```bash
   dotnet run
   ```
   После запуска вы увидите адрес: `http://localhost:5149`

5. **Откройте фронтенд**
   - Перейдите по адресу `http://localhost:5149` — вы попадёте на главную страницу (файл `wwwroot/index.html`).

## 🧠 Логика приоритетов

Вес задачи = **Сложность** + **Срочность** (каждая от 1 до 3).

- **5–6** → Красный 🔴 (высокий приоритет)
- **3–4** → Жёлтый 🟡 (средний приоритет)
- **2** → Зелёный 🟢 (низкий приоритет)

Цвет фона карточки меняется соответственно, что позволяет мгновенно оценить важность задачи.

## 🔄 Автономный режим и очистка данных

- Все дополнительные данные хранятся в **`sessionStorage`**.
- При перезагрузке страницы данные восстанавливаются.
- При **закрытии вкладки** все задачи, приоритеты, таймеры и дедлайны **автоматически удаляются**.
- Если нужно сохранить URL API (например, при переходе на продакшн), используется `localStorage` отдельно.
